\# AI Workflow Comparison



For this exercise, I built the same settings page twice using two different AI prompting approaches.



\## Round 1 – Vague Prompt



I used a simple prompt asking the AI to create a settings page with validation. The generated code worked partially, but it required manual review. Some improvements were missing, such as clearer validation behavior, better accessibility, and cleaner project organization. I accepted the output with minimal guidance to observe the quality of a vague prompt.



\## Round 2 – Precise Prompt



In the second round, I provided detailed requirements, including file names, HTML, CSS, and JavaScript constraints, accessibility requirements, validation rules, and a request for self-review. The generated code was better structured and easier to understand. Labels, validation messages, theme switching, and code organization improved significantly.



\## Comparison



The precise prompt produced cleaner code, better accessibility, and more consistent validation. The project structure was clearer because HTML, CSS, and JavaScript were separated correctly. The code also required less manual cleanup compared to the first version.



\## AI Mistake I Found



One issue I identified was that the Save button behavior was not fully working as expected even though validation and theme switching worked correctly. This required manual testing and additional debugging, showing that AI-generated code should always be verified before being accepted.



\## Conclusion



This exercise demonstrated that the quality of AI-generated code depends heavily on the quality of the prompt. A detailed prompt combined with verification and manual testing produces more reliable, maintainable, and accessible software than a vague prompt.
